# Multi-Model Analysis 演进历史

## 概述

Hermes Agent 的多模型分析系统经历了两个主要阶段的演进，从简单的多模型调用发展为完整的协作方法论。

## v1.0: multi-model-analysis（第一版）

### 时间线
- **创建时间**：2026年6月5日
- **核心文件**：`src/skill-v1.md`

### 设计理念
通过 API 平行调用多个不同模型，分别读取各自输出，再合成统一分析结论。

### 核心功能
1. **多模型调用** - 支持 OpenRouter 上的多个模型
2. **并行执行** - 同时调用多个模型
3. **自动合成** - 找共性、找差异、找缺席

### 模型阵容
- `deepseek/deepseek-chat` (V3) - 文学批判风格
- `deepseek/deepseek-v3.2` - 结构分析风格
- `deepseek/deepseek-r1` - 激进批判风格
- `qwen/qwen3-235b-a22b` - 中国对齐风格
- `qwen/qwen3-235b-a22b-thinking-2507` - 思辨批判风格

### 局限性
- 合成逻辑较简单
- 缺乏系统化的 prompt 模板
- 没有信息图管道

## v2.0: multi-model-collaboration（协作版）

### 时间线
- **创建时间**：2026年6月5日
- **核心文件**：`src/skill-v2.md`

### 设计理念
系统化的多模型协作方法论，提供更完善的文档和工具。

### 核心改进

#### 1. 模型选择策略
- **多样性优先** - 选择不同家族的模型
- **分层架构** - 主力层 + 备选层 + 最小可行集
- **动态调整** - 根据可用性自动降级

#### 2. Prompt 模板
- **统一格式** - 所有模型使用相同 prompt
- **多维分析** - 认知/社会/权力三个维度
- **输出规范** - 统一字数和格式要求

#### 3. 合成方法
- **找共性** - 跨模型共识
- **找差异** - 独特视角
- **找缺席** - 补充视角

#### 4. 信息图管道
- **可视化** - 支持生成信息图
- **多模态** - 文本 + 图表结合

### 改进对比

| 特性 | v1.0 | v2.0 |
|------|------|------|
| **模型选择** | 固定列表 | 动态策略 |
| **Prompt 模板** | 简单 | 系统化 |
| **合成方法** | 基础 | 多维分析 |
| **信息图** | 不支持 | 支持 |
| **文档质量** | 基础 | 完善 |

## 多模型路由策略

### 时间线
- **创建时间**：2026年6月
- **核心文件**：`src/routing-strategy.md`

### 设计理念
根据任务复杂度自动选择最优模型，平衡推理深度和运行经济性。

### 核心功能

#### 1. 任务分类
- **有图片/视频/音频** → MiMo-V2-Omni
- **长文本/文档** → MiMo-V2.5
- **复杂推理** → MiMo-V2.5-Pro
- **日常闲聊** → DeepSeek V4 Flash
- **简单任务** → DeepSeek V4 Flash

#### 2. 成本优化
- **MiMo** - 中国区直连，成本低
- **DeepSeek** - 轻量级任务，更便宜
- **OpenRouter** - 特殊模型，按需使用

#### 3. 降级策略
- 主模型不可用 → 自动切换到辅助模型
- API 超时 → 重试或降级
- 成本超限 → 切换到更便宜的模型
