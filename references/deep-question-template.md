# 深度分析 Prompt 模板

## 结构要素

这个模板适用于需要多模型协作进行深入分析的问题。

```
# 问题标题

## 背景（2-4句话说明上下文）
[简洁描述场景/系统/工具的背景。让模型不需要任何外部知识也能理解。]

## 请从这个角度深度分析
[核心问题。要具体，有方向性。不要泛泛问"你怎么看"——要给定分析框架。]

请从三个层面分析：
1. **[层面A]**：[具体分析角度]
2. **[层面B]**：[具体分析角度]
3. **[层面C]**：[具体分析角度]

## 格式要求
- 用中文回答
- 约 800-1000 字
- 要有锐度，不要安全废话
- [其他要求，如：分成三个section、每节有小标题]
```

## 示例（来自实际案例）

见多模型分析"Skills Judgment 熵减系统"的成功案例。

## 提示

- 背景信息要足够让不同训练范式的模型都能理解问题
- "三个层面"框架产生结构化输出，便于后续合成
- 不要写"不要安全废话"——某些模型的安全 filter 会拒答
- 温度用 0.7 比较合适：平衡多样性和连贯性
